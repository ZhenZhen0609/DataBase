DBMS 第四阶段（DDL简单解析特别档）

这次任务的逻辑非常纯粹：“用户打字 -> 橙圈拆解指令 -> 红圈根据指令干活 -> 粉圈展示结果”。
以下是针对“DDL 简单解析”任务的详细拆解：

1. 核心逻辑链条：
用户输入：CREATE TABLE students (id INT, name TEXT)
1.粉圈 (D) 拿到这串话，交给橙圈。
2.橙圈 (B) 把这串话切开，翻译成：动作=CREATE TABLE，名字=students，字段=id(INT), name(TEXT)。
3.橙圈 (B) 调用红圈的函数。
4.红圈 (A) 去硬盘上建文件。

🟠 橙圈 (B) 的详细任务：编写 SQL 翻译机 (SQLParser)
你的任务： 你是整个系统的“大脑翻译官”，要把字符串变成具体的指令。
第一步：分词 (Tokenize)
o创建一个函数 void parseSQL(QString sql)。
o先去掉末尾的分号 ;。
o用 QStringList list = sql.split(" "); 把句子按空格拆开。
第二步：逻辑分支 (Branching)
o分支1：数据库操作
如果 list[0] 是 CREATE 且 list[1] 是 DATABASE：
提取出 list[2]（数据库名）。
联动： 调用 red_A->createDatabase(list[2]);
o分支2：表操作 (稍微难一点)
如果 list[0] 是 CREATE 且 list[1] 是 TABLE：
提取出 list[2]（表名）。
解析括号里的字段： 找到 ( 和 ) 之间的内容，按逗号 , 拆开，再按空格拆开得到字段名和类型。
联动： 把拆好的名字和字段列表传给红圈：red_A->createTable(dbName, tableName, fields);
第三步：反馈
o如果解析成功，返回 Success；如果用户写错了（比如写成 CREAT），返回 Syntax Error（语法错误）。

🔴 红圈 (A) 的详细任务：编写执行引擎 (Execution)
你的任务： 你不需要管用户输入了什么，你只需要保证橙圈 B 给你的参数，你能精准执行。
核心工作：
1.准备好被调用： 确保你的 createDatabase(QString name) 和 createTable(QString db, QString table, QList<Field> fields) 函数已经完全写好并且能稳定运行。
2.错误处理： 如果橙圈让你建一个已经存在的数据库，你要返回 false，并告诉橙圈为什么失败。
3.支持删除： 同样准备好 dropDatabase 和 dropTable 的函数，等橙圈解析出 DROP 指令时调用你。
联动关键： 你的函数参数必须和橙圈解析出来的东西对得上（名字对名字，列表对列表）。

💗 粉圈 (D) 的详细任务：编写 SQL 终端 (Terminal UI)
你的任务： 做一个像黑窗口一样的输入区，并把结果实时展示出来。
第一步：UI 组件
o在现有界面的下方（或者单独一个 Tab）加一个 QLineEdit（输入框）和一个“执行”按钮。
o在旁边加一个“清空日志”按钮。
第二步：指令下发
o点击“执行”时，读取输入框里的内容。
o联动： 调用 orange_B->parseSQL(text);。
第三步：自动刷新 (非常重要！)
o如果橙圈反馈执行成功了，不要让用户手动点刷新。
o你的程序要自动调用 refreshTree()，让用户立刻看到左侧树状图多了一个图标。
o演示效果： 用户回车一敲，树状图瞬间更新，这在答辩时是巨大的加分项！

🤝 三人如何联动的代码示意：

QString cmd = ui->sqlInput->text();

    ui->logOutput->append("执行成功！");
    refreshTree(); // 自动刷新界面

    ui->logOutput->append("错误：" + res.message);
}
